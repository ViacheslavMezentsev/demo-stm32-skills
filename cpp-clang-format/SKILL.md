# Навык форматирования C/C++ кода

Этот навык описывает правила применения конфигурации `.clang-format` для проектов на C и C++. Конфигурация является "Единственным Источником Правды" для стиля кода.

## Основные правила

1. **Формат**: Файл `.clang-format` должен находиться в корневой директории проекта.
2. **Применение**: Используйте `clang-format -i <файл>` для форматирования файлов на месте.
3. **Проверка**: Используйте `clang-format --dry-run <файл>` для проверки стиля без изменений.
4. **Интеграция**: Настройки автоматически подхватываются IDE (VS Code, CLion, Visual Studio) при наличии файла.

## Ключевые отличия от дефолтных настроек

Данная конфигурация содержит специфические правила, которые **отличаются** от стандартного стиля LLVM или Google:

| Параметр | Значение | Описание |
| --- | --- | --- |
| `ColumnLimit` | `120` | Увеличенная ширина строки (стандарт обычно 80) |
| `IndentWidth` | `4` | Отступы 4 пробела (не табы) |
| `UseTab` | `Never` | Запрет на использование символов табуляции |
| `BreakBeforeBraces` | `Custom` | Пользовательская логика расположения скобок |
| `BraceWrapping` | `AfterClass: true` | Скобки классов/структур на новой строке |
| `BraceWrapping` | `AfterFunction: true` | Скобки функций на новой строке |
| `SpaceAfterCStyleCast` | `true` | Пробел после C-стиля приведения типа |
| `SpacesInParentheses` | `true` | Пробелы внутри круглых скобок `( x )` |
| `Standard` | `Cpp11` | Стандарт языка для парсинга |

## Примеры использования

### Форматирование конкретного файла
```bash
clang-format -i src/main.cpp
```
